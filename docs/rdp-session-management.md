# Управление RDP-сессиями: стабильная работа на ALT Linux + xrdp

**Версия:** 1.0
**Окружение:** ALT Workstation K 10.4, xrdp 0.10.2-alt2, KDE Plasma 5
**Дата:** 2026-02-18

---

## Проблемы

### 1. При повторном подключении открывается новая сессия вместо существующей

При подключении с другого ПК (или с тем же ПК, но с иным разрешением экрана) xrdp создаёт
новую KDE-сессию вместо присоединения к уже запущенной.

**Причина:** Параметр `Policy=Default` в `/etc/xrdp/sesman.ini` соответствует режиму `UB`
(User + Bits-per-pixel). Это означает, что сессия идентифицируется по сочетанию
пользователь + глубина цвета. При подключении с экрана с другим разрешением или bpp
система видит запрос как запрос на отдельную сессию.

### 2. Сессия завершается при разрыве соединения (приложение "не открывается")

После потери RDP-соединения (разрыв сети, закрытие клиента) при следующем подключении
сессия не восстанавливается — KDE уже завершил работу.

**Причина:** Драйвер `xorg-drv-xrdp` завершает X-сессию при `idle_disconnect_timeout=0`
(лог: `rdpClientConInit: disconnect idle session after [0] sec`). Сессия прекращается
вместо того чтобы оставаться в отключённом состоянии.

---

## Диагностика

Проверить текущее состояние сессий:

```bash
# Активные xrdp-сессии
systemctl status xrdp-sesman

# Логи сессий (последние события)
grep -E "session|Session|IP:|display" /var/log/xrdp-sesman.log | tail -20

# Текущая политика
grep "^Policy" /etc/xrdp/sesman.ini

# Статус KillDisconnected
grep "KillDisconnected" /etc/xrdp/sesman.ini
```

---

## Решение

### Шаг 1. Изменить политику сессий в sesman.ini

Файл: `/etc/xrdp/sesman.ini`

```ini
[Sessions]
# Было:
Policy=Default

# Стало — сессии разделяются только по пользователю,
# переподключение с любого ПК заходит в ту же сессию:
Policy=U

# Убедиться что эти параметры установлены:
KillDisconnected=false
DisconnectedTimeLimit=0
IdleTimeLimit=0
```

Применить через crudini (идемпотентно):

```bash
crudini --set /etc/xrdp/sesman.ini Sessions Policy U
crudini --set /etc/xrdp/sesman.ini Sessions KillDisconnected false
crudini --set /etc/xrdp/sesman.ini Sessions DisconnectedTimeLimit 0
```

> **Внимание:** xrdp-patch-6-rtk при переустановке перезаписывает sesman.ini.
> Повторно применять эти команды после обновления пакета `xrdp-patch`.

### Шаг 2. Настроить восстановление KDE-сессии

Чтобы KDE восстанавливал открытые приложения при переподключении:

```bash
# Применить для пользователя (выполнить под нужным пользователем):
kwriteconfig5 --file ksmserverrc --group General --key loginMode restorePreviousLogout
```

Или вручную в файле `~/.config/ksmserverrc`:

```ini
[General]
loginMode=restorePreviousLogout
```

### Шаг 3. Перезапустить xrdp-sesman

```bash
systemctl restart xrdp-sesman
```

> Текущие активные сессии будут завершены. Выполнять при отсутствии активных пользователей.

---

## Проверка результата

После перезапуска:

1. Подключиться по RDP с первого ПК — сессия создаётся (`:10`)
2. Отключить RDP-клиент (не завершать сессию)
3. Подключиться по RDP со второго ПК с тем же пользователем
4. Ожидаемый результат: переподключение к существующей сессии на `:10`, приложения сохранены

Проверить в логе:

```bash
grep "session" /var/log/xrdp-sesman.log | tail -10
# Должно быть: "found existing session" или переподключение к display :10
# Не должно быть: "create a session" при наличии уже запущенной
```

## Пояснение к значениям Policy

| Значение | Поведение | Рекомендация |
|----------|-----------|--------------|
| `Default` | UB: user + bpp | Не рекомендуется при подключении с разных ПК |
| `U` | Только по пользователю | **Рекомендуется** для одного пользователя с разных устройств |
| `UB` | User + bits-per-pixel | Создаёт отдельные сессии при разном цветовом режиме |
| `UBD` | + display size | Создаёт отдельные сессии при разном разрешении |
| `UBDI` | + IP-адрес | Каждое подключение с нового IP — новая сессия |
| `Separate` | Всегда новая сессия | Не подходит для переподключения |

---

## Ограничения и известные особенности

- **KillDisconnected=false** + `Policy=U`: незавершённые сессии накапливаются при выходе через
  кнопку "Отключить" без явного logout. Рекомендуется настроить мониторинг или периодическую
  очистку через `DisconnectedTimeLimit` (например, `86400` = 24 часа).

- **xrdp-patch-6-rtk** перезаписывает `/etc/xrdp/sesman.ini` и `/etc/pam.d/xrdp-sesman` при
  переустановке. Изменения из Шага 1 необходимо повторять после обновления этого пакета.

- **KDE + XRDP**: systemd пользовательской сессии недоступен в xrdp-сессии
  (`org.freedesktop.systemd1 failed: exit status 1` в логах — нормально).

- **RAM**: xrdp-sesman показывает ~7GB использования памяти — это cgroup KDE Plasma
  (startplasma-x11 + все KDE-сервисы), не утечка xrdp.

## Связанные файлы

| Файл | Назначение |
|------|-----------|
| `/etc/xrdp/sesman.ini` | Политика сессий, таймауты |
| `/etc/xrdp/xrdp.ini` | Основной конфиг xrdp (сжатие, безопасность) |
| `/etc/xrdp/startwm.sh` | Запуск KDE Plasma при новой сессии |
| `/etc/xrdp/reconnectwm.sh` | Скрипт при переподключении (пустой) |
| `/var/log/xrdp-sesman.log` | Лог сессий (создание, завершение, ошибки) |
| `/var/log/xrdp.log` | Лог xrdp демона (подключения) |
